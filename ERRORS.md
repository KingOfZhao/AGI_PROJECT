# ERRORS.md — 错误与教训记录

> 每次失败/报错/用户纠正后，记录到这里
> 格式：[日期] 问题描述 → 根因 → 教训 → 防止复发

---

## 2026-03-30

### E001: BrowseComp搜索全失败
- **问题**: DuckDuckGo API、Google、Bing国内版全部不可用
- **根因**: 网络环境限制，未提前验证搜索可用性
- **教训**: 先验证工具链再开始任务，不假设任何外部服务可用
- **防止**: 每次使用新工具前写smoke test

### E002: SCP部署失败（Permission denied）
- **问题**: 没有服务器SSH密钥，代码改了但部署不了
- **根因**: 之前会话配置了SSH但没有持久化密钥
- **教训**: 基础设施（SSH/部署通道）优先于功能开发
- **防止**: 确认部署通道可用再写代码

### E003: 用户批评"产出太少"
- **问题**: 花大量时间在BrowseComp自测上，实际代码产出少
- **根因**: 把"自进化"当成了目标，而不是手段。自测自嗨不等于产出
- **教训**: 产出 > 自评。用户要的是可交付成果，不是自测分数
- **防止**: 每个工作时段必须有≥1个可交付物（代码/文档/部署）

### E004: browser-use云服务需要API Key
- **问题**: 安装了browser-use但本地模式需要API Key
- **根因**: 没有提前调研本地模式的配置方式
- **教训**: 安装前先查文档确认使用方式
- **防止**: 新工具安装前5分钟文档调研

### E005: bash for循环JSON解析错误
- **问题**: for循环里$()语法导致CRM批量更新400
- **根因**: bash变量展开在循环中行为不一致
- **教训**: 批量API调用用Python脚本，不用bash
- **防止**: 超过3次API调用用Python

### E006: Chart.js加载顺序导致运行时错误
- **问题**: script标签放在自定义JS之后，首次调用时Chart未定义
- **根因**: 没有考虑外部库的加载时序
- **教训**: 第三方库script标签必须放在使用它的代码之前
- **防止**: 所有CDN库统一放在<head>中或第一个<script>之前
- **发现方式**: 伟人竞技场审查者模式（独立审查发现）
