## 一、Pacdora 120370 模型逆向工程分析

### 1.1 3D模型结构 (从 demoProjectData JSON 逆向)

Pacdora 使用 **Three.js Object3D.toJSON** (v4.5) 序列化完整3D场景，核心发现:

#### 几何体: 全部使用 ExtrudeGeometry

```
所有面板均使用 ExtrudeGeometry:
- shapes: 2D Path 定义面板轮廓 (LineCurve + EllipseCurve)
- options.depth: 0.5  ← 材料厚度 0.5mm
- bevelEnabled: false ← 无倒角
```

**关键点**: Pacdora 的每个面板都是 **带厚度的挤出体**，不是0厚度面片。这正是我们推演报告中 R3 反贼(3D渲染无厚度)指出的 DiePre AI 缺陷。

#### 折叠层级树 (24个面板)

```
B (底板, 主面板)
├── B_BL (底板→左侧壁 铰链)
│   └── BL (左侧壁)
│       ├── BL_BL1 (左侧壁→左侧壁折耳1 铰链)
│       │   └── BL1 → BL1_BL2 → BL2 (三段折叠)
│       ├── BL_FL1 (左侧壁→前折耳 铰链)
│       │   └── FL1 (前左折耳)
│       └── BL_HL1 (左侧壁→后折耳 铰链)
│           └── HL1 (后左折耳)
├── B_BR (底板→右侧壁 铰链)
│   └── BR (右侧壁)
│       ├── BR_BR1 → BR1 → BR1_BR2 → BR2 (三段折叠)
│       ├── BR_FR1 → FR1 (前右折耳)
│       └── BR_HR1 → HR1 (后右折耳)
├── B_F (底板→前壁 铰链)
│   └── F (前壁)
│       ├── F_FR → FR (前右折耳)
│       ├── F_FL → FL (前左折耳)
│       └── F_FT → FT (前壁顶部折叠)
│           ├── FT_FT1 → FT1 (顶部延伸)
│           ├── FT_FTR → FTR (顶右折耳)
│           └── FT_FTL → FTL (顶左折耳)
└── B_H (底板→后壁 铰链)
    └── H (后壁)
        ├── H_HR → HR (后右折耳)
        ├── H_HL → HL (后左折耳)
        └── H_HB → HB (后壁底部折叠)
            ├── HB_HB1 → HB1 (底部延伸)
            ├── HB_HBL → HBL (底左折耳)
            └── HB_HBR → HBR (底右折耳)
```

**总计: 24个面板 + 24个铰链节点 = 48个 Three.js 对象**

#### 铰链实现方式

每个铰链是一个独立的 ExtrudeGeometry mesh:
- **Shape**: 小圆 (EllipseCurve, r=0.26mm)
- **ExtrudePath**: LineCurve3 沿折线方向挤出
- 挤出长度 = 面板边长 (例如底板到侧壁铰链长度 = 100.75mm ≈ W/2)

```json
{
  "type": "ExtrudeGeometry",
  "shapes": ["圆形截面 r=0.26mm"],
  "options": {
    "extrudePath": {
      "type": "LineCurve3",
      "v1": [0.05, -100.75, 0],
      "v2": [-0.05, 100.75, 0]
    }
  }
}
```

#### 材质系统

```json
{
  "outsideMaterial": {
    "type": "MeshPhysicalMaterial",
    "roughness": 1,
    "metalness": 0,
    "clearcoat": 0,
    "map": "outsideCanvasTexture"  // 外表面贴图(用户设计面)
  },
  "insideMaterial": {
    "type": "MeshPhysicalMaterial",
    "roughness": 1,
    "metalness": 0,
    "map": "insideCanvasTexture"  // 内表面贴图
  }
}
```

**每个面板使用3个材质**: `[outsideMaterial, insideMaterial, outsideMaterial]` — 对应挤出体的正面、背面、侧面。

#### 折叠角度 (从旋转矩阵提取)

| 铰链 | 旋转轴 | 角度 | 含义 |
|------|--------|------|------|
| B→BL | Z轴 | -π/2 | 左侧壁完全折起90° |
| B→BR | Z轴 | +π/2 | 右侧壁完全折起90° |
| B→F | Y轴 | -π/2 | 前壁完全折起90° |
| B→H | Y轴 | +π/2 | 后壁完全折起90° |
| BL→BL1 | 反向微调 | -0.0411rad ≈ -2.35° | 侧壁折耳微弯 |
| F→FT | X轴 | -0.0349rad ≈ -2° | 前壁顶部微向内弯 |
| FT→FT1 | X轴 | +0.0140rad ≈ 0.8° | 顶部延伸微弯 |
| BL→FL1/HL1 | 近-90° | -0.00349rad偏移 | 折耳向内折叠 |

**关键发现**: 折叠角度不是简单的0°/90°，而是每个铰链有独立的精确角度，包括微小弯曲(±2°)来模拟纸板的自然弯曲形态。

#### 四折斜角 (Mitered Corners)

120370是 "四折斜角" 盒型，其核心特征是角部的**三角形斜切面**:

```
形状示例 (左前角):
  v1: [120.7, 70.1]     ← 侧壁与前壁交点
  v2: [120.7, 120.7]    ← 角顶点
  v3: [77.26, 77.26]    ← 斜切45°线上的点
```

这是一个等腰直角三角形，45°斜切线将正方形角部一分为二，实现四折托盘的斜角结构。4个角各有2个三角形面板（共8个三角形）。

#### 光照系统

```json
{
  "AmbientLight": { "intensity": 0.7 },
  "DirectionalLight": { "intensity": 0.35, "position": [-20, 20, 20], "shadow": true }
}
```

### 3.2 新发现反贼详细分析

#### R24: 3D几何体不精确 (致命)

**Pacdora实现**:
```javascript
// Pacdora: 精确2D Shape → ExtrudeGeometry
const shape = new THREE.Shape()
shape.moveTo(121.2, 20)
shape.lineTo(122.2, 20)
shape.lineTo(319.8, 20)
shape.lineTo(320.8, 20)
shape.lineTo(320.8, 69.6)
// ... EllipseCurve圆角
const geometry = new THREE.ExtrudeGeometry(shape, { depth: 0.5 })
```

**DiePre AI现状**:
```javascript
// DiePre: 矩形近似
const geo = new THREE.BoxGeometry(w, h, t)  // 只能矩形
```

**差距**: Pacdora 每个面板的2D轮廓都是精确的Shape路径(含直线+圆弧)，挤出后得到真实形状。DiePre 只用BoxGeometry，无法表达梯形折耳、三角形斜角、圆角连接等异形面板。

**镇压方案**: 
1. 为每种FEFCO编码定义参数化Shape生成函数
2. 用 `ExtrudeGeometry(shape, {depth: materialThickness})` 替代 `BoxGeometry`
3. Shape中包含LineCurve(直线) + EllipseCurve(圆角) 路径

#### R25: 折叠层级只有1级 (致命)

**Pacdora实现**: 
```
B → B_BL铰链 → BL → BL_BL1铰链 → BL1 → BL1_BL2铰链 → BL2
```
最深3级嵌套: 底板→侧壁→折耳1→折耳2

**DiePre AI现状**: 
```
scene → pivotFront → frontPanel (仅1级)
```
底板→壁，无折耳/粘口层级

**镇压方案**: 
1. 构建折叠树数据结构: `{panel: 'B', children: [{panel: 'BL', hinge: {axis:'Z', pos:[120.2, 221.5]}, children: [...]}]}`
2. 递归创建 `THREE.Group` 嵌套，每级Group以铰链位置为旋转中心
3. 折叠动画: 递归设置每级Group的rotation

#### R26: 无异形面板(斜角/梯形) (致命)

**Pacdora的四折斜角**:
```javascript
// 45°三角形斜切面板
shape.moveTo(120.7, 70.1)
shape.lineTo(120.7, 120.7)
shape.lineTo(77.26, 77.26)  // 45°线
shape.lineTo(120.7, 70.1)   // 闭合
```

**镇压方案**:
1. 在FEFCO模板引擎中定义各盒型的角部处理方式
2. 支持: 直角(90°正交)、斜切(45° mitered)、圆角、锁底等
3. 角部Shape由模板函数根据L/W/H自动计算

#### R27: 材质渲染不真实 (严重)

|------|--------------|-----------|
| 类型 | MeshPhysicalMaterial | MeshPhongMaterial |

| 金属度 | metalness: 0 | N/A |

// 升级为PBR材质
const material = new THREE.MeshPhysicalMaterial({

#### R30: 折叠角度单一 (严重)

**Pacdora**: 每个铰链有独立的精确角度:
- 主壁: ±π/2 (90°)
- 折耳: -0.0411rad (-2.35°) 微弯
- 顶板: -0.0349rad (-2°)

**DiePre AI**: `foldRad = (props.foldAngle / 90) * (Math.PI / 2)` — 全局单一角度

**镇压方案**:
```javascript
// 每个铰链独立角度
const foldConfig = {
  'B→F': { axis: 'x', angle: -Math.PI/2, springback: 0 },
  'B→H': { axis: 'x', angle: Math.PI/2, springback: 0 },
  'F→FT': { axis: 'x', angle: -0.0349, springback: 0.002 },
  'BL→BL1': { axis: 'z', angle: -0.0411, springback: 0.003 },
  // ...
}
```

#### R32: 2D刀线无法自动生成 (致命)

**Pacdora**: 用户输入L/W/H → 系统自动生成完整2D刀线图(含折痕位置、粘口位置、斜角切割线、尺寸标注)

**DiePre AI**: 用户需手动绘制或导入DXF

**镇压方案**:
1. 为每种FEFCO编码创建参数化2D布局函数
2. 输入: `{L, W, H, materialThickness, cornerType}` 
3. 输出: `{entities: [{type:'LINE', role:'cut', ...}, {type:'LINE', role:'crease', ...}], bounds: {x, y, w, h}}`

```python
# 120370 托盘盒四折 2D布局生成伪代码
def generate_tray_four_fold(L, W, H, t):
    entities = []
    # 底板 B: (H+margin, H+margin) to (H+margin+L, H+margin+W)
    bx, by = H + 20, H + 20  # 20mm = 折耳宽度
    
    # 底板切割线
    entities.append(rect(bx, by, L, W, role='cut'))
    
    # 四面壁折痕线
    entities.append(line(bx, by, bx+L, by, role='crease'))          # 前壁折痕
    entities.append(line(bx, by+W, bx+L, by+W, role='crease'))      # 后壁折痕
    entities.append(line(bx, by, bx, by+W, role='crease'))           # 左壁折痕
    entities.append(line(bx+L, by, bx+L, by+W, role='crease'))      # 右壁折痕
    
    # 四面壁面板
    entities.append(rect(bx, by-H, L, H, role='cut'))               # 前壁
    entities.append(rect(bx, by+W, L, H, role='cut'))               # 后壁
    entities.append(rect(bx-H, by, H, W, role='cut'))               # 左壁
    entities.append(rect(bx+L, by, H, W, role='cut'))               # 右壁
    
    # 四个角的45°斜切三角形
    for corner in ['FL', 'FR', 'HL', 'HR']:
        entities.append(triangle_45(corner, bx, by, L, W, H, role='cut'))
    
    # 折耳(梯形)
    for ear in ['top', 'bottom', 'left_top', 'left_bottom', ...]:
        entities.append(trapezoid_ear(ear, bx, by, L, W, H, role='cut'))
    
    return entities
```

### 7.10 批量获取策略建议

```python
import requests, json, time

# 策略1: 批量下载3D模型JSON (1579个, 无需鉴权)
for m in models:
    url = m.get("demoProjectDataUrl")
    if url:
        resp = requests.get(url)
        with open(f"3d_models/{m['num']}.json", "w") as f:
            f.write(resp.text)
        time.sleep(0.5)  # 礼貌间隔

# 策略2: 批量下载2D刀线图PNG (5910个)
for m in models:
    knife = m.get("knife")
    if knife:
        resp = requests.get(knife)
        with open(f"dielines/{m['num']}.png", "wb") as f:
            f.write(resp.content)
        time.sleep(0.3)

# 策略3: 批量下载材质预览图
for m in models:
    for ms in m.get("modeSetting", []):
        img = ms.get("image")
        if img:
            resp = requests.get(img)
            with open(f"materials/{m['num']}_{ms['material']}.png", "wb") as f:
                f.write(resp.content)
            time.sleep(0.3)
```

**注意**: 
- 3D JSON 和 静态图片均为 **CDN直连，无鉴权**
- SPA页面(刀线详情/样机详情)需 **Headless Chrome/Playwright** 才能获取完整交互内容
- 编辑器功能需 **登录 Pacdora 账号** 才能使用

### 7.11 关键发现总结

1. **3D模型数据可直接获取**: `https://cloud.pacdora.com/demoProject/{num}.json` — 无鉴权，1579个模型有3D JSON
2. **2D刀线图可直接获取**: `https://oss.pacdora.cn/preview/dieline-{num}.png` — 99%覆盖率
3. **URL构造规则已确认**: `nameKey` 字段直接作为详情页路径，5967/5969 模型的 `nameKey` 已包含 `num`
4. **可调节内容来源**: 尺寸(L/W/H) + 材质(modeSetting) + 3D场景(demoProjectDataUrl JSON) + 在线设计(需SPA交互)
5. **SPA限制**: 页面交互内容(折叠动画/尺寸调节/材质切换/在线设计)无法通过静态抓取获得，需 Headless Browser
6. **编辑器入口**: 详情页description描述"点击'编辑'按钮，可对{name}进行在线编辑，模板、尺寸和材质可自定义修改"
