## Round 11/20 — AI结构生成引擎: 从"选模板"到"描述→生成"

### 核心突破: 消灭模板

当前Pacdora和DiePre都是**模板驱动**: 用户从5969个预定义盒型中选择。
这是工业时代的思维。真正的突破是:

**用户说: "我需要一个能装6瓶红酒、带提手、要能堆叠的盒子"**
**系统输出: 完整的3D结构 + 2D刀线 + 工艺参数**

### 架构: StructureGPT

```
用户意图 (自然语言/草图/参考图)
  ↓
IntentParser (LLM)
  → 解析出: 内容物(6瓶红酒, ∅75mm×300mm)
  → 功能需求: 提手, 可堆叠, 分隔
  → 约束: 瓦楞纸, 成本<¥15
  ↓
StructureReasoningEngine
  → 从知识图谱检索: 酒盒→分隔结构→提手类型→堆叠强度
  → 碰撞: 6瓶排列(2×3 vs 3×2 vs 1×6)
  → 每种排列 → 计算最优外形尺寸
  → 生成候选结构: 3-5个方案
  ↓
ParametricStructureGenerator
  → 对每个方案:
    → 生成FoldTree (不是查表, 是从约束求解)
    → Shape路径由几何约束方程组解出
    → 折叠角度由材料力学决定(不是预设)
  ↓
RankingEngine
  → 材料成本 × 强度评分 × 生产效率 × 美观度
  → 输出TOP3方案供用户选择
```

### 关键技术: 约束求解器

```python
class StructureSolver:
    """从功能约束生成几何结构 — 不依赖模板"""
    
    def solve(self, constraints):
        # 1. 内容物包络体
        envelope = self.compute_envelope(constraints.contents)
        # [6瓶酒] → 包络体 240mm × 160mm × 310mm
        
        # 2. 结构类型推理
        structure_type = self.infer_structure(
            envelope, constraints.functions)
        # 需要分隔 + 提手 → 推理出: 开窗提手盒+十字隔板
        
        # 3. 壁面方程组
        # 每个壁面是一个约束方程:
        # - 几何约束: 相邻壁面共享边
        # - 力学约束: 壁面承重 ≥ 堆叠载荷/面积
        # - 材料约束: 弯折半径 ≥ 2×厚度
        # - 制造约束: 最小折耳宽度 ≥ 15mm
        walls = self.solve_wall_equations(envelope, structure_type)
        
        # 4. 连接拓扑
        # 自动推理哪些壁面需要连接、用什么方式
        # 可选: 折叠连接 / 粘合 / 插口 / 锁扣
        topology = self.solve_connections(walls, constraints)
        
        # 5. 展开路径求解
        # 从3D拓扑→2D展开图 (最小废料率)
        unfolding = self.solve_unfolding(walls, topology)
        
        return Structure(walls, topology, unfolding)
```

## Round 12/20 — 几何约束语言: 包装领域的"编程语言"

### 问题: 为什么需要新语言?

当前所有包装CAD软件(ArtiosCAD/PackFlow/Pacdora)都用**硬编码函数**定义盒型。
每种新盒型需要程序员写代码。5969个模板 = 5969个函数。

我们需要的是一种**声明式约束语言**, 让任何人(甚至AI)可以"描述"一种盒型:

### BoxScript: 包装结构描述语言

```boxscript
// 定义一个从未存在过的盒型: 六角形抽屉盒
box HexDrawer {
  // 声明面板
  panel bottom: hexagon(side=L/2)
  panel wall[6]: rect(width=L/2, height=H)
  panel drawer_bottom: hexagon(side=L/2 - t*2)
  panel drawer_wall[6]: rect(width=L/2 - t*2, height=H*0.8)
  panel lid: hexagon(side=L/2 + 2)  // 略大于底, 便于盖合
  
  // 声明连接
  fold bottom.edge[i] -> wall[i] at 90°
  fold drawer_bottom.edge[i] -> drawer_wall[i] at 90°
  
  // 声明约束
  constraint: wall[i].width == bottom.edge[i].length
  constraint: sum(wall[i].thickness) < L  // 不能太厚合不上
  constraint: drawer fits_inside box with clearance=1mm
  
  // 功能声明
  feature: pull_tab on drawer_wall[0], width=30mm
  feature: finger_hole on wall[0], radius=15mm
  
  // 材料
  material: FLUTE for box, WHITE_BOARD for drawer
}
```

### 编译器: BoxScript → FoldTree + Entities2D

```
BoxScript源码
  ↓ Parser (PEG语法)
AST (抽象语法树)
  ↓ ConstraintResolver
  │  → 求解所有 constraint 方程
  │  → 得到具体数值几何
  ↓ TopologyBuilder  
  │  → 从 fold 声明构建折叠拓扑图
  │  → 自动计算铰链位置和旋转轴
  ↓ UnfoldingSolver
  │  → 从拓扑图求解最优2D展开
  │  → NP-hard → 启发式/beam search
  ↓ CodeGen
  │  → 输出 FoldTree JSON (给3D引擎)
  │  → 输出 entities2D (给2D引擎)
  │  → 输出 DXF (给生产)
```

### 为什么这是颠覆性的

1. **AI可以写BoxScript** — LLM输出结构化文本远比输出3D JSON容易
2. **可验证** — 约束求解器自动验证结构可行性
3. **可组合** — `import handle from "standard/handle"` 模块化复用
4. **可版本控制** — 纯文本, git diff友好
5. **无限盒型** — 不需要程序员为每种盒型写函数

## Round 13/20 — 物理仿真引擎: 从"动画折叠"到"真实折叠"

### 当前的假: 所有3D折叠都是动画

Pacdora的折叠: 旋转Group.rotation → 纯几何变换, 不考虑物理
DiePre的折叠: 同上

**真实纸板折叠**:
- 折痕两侧纤维断裂, 形成V槽
- 弯曲半径 = f(厚度, 压痕深度, 纤维方向)
- 回弹角 = f(纤维弹性模量, 弯曲角度, 时间)
- 多层瓦楞: 面纸和芯纸变形不同步
- 含水率变化 → 尺寸膨胀 → 折叠角度漂移

### FoldPhysicsEngine

```python
class FoldPhysicsEngine:
    """基于材料力学的真实折叠仿真"""
    
    def simulate_fold(self, fold_edge, material, angle_target):
        t = material.thickness  # mm
        E = material.elastic_modulus  # MPa (纸板: 2000-8000)
        crease_depth = material.crease_depth_ratio * t  # 压痕深度
        
        # 1. 弯曲力矩
        # M = E × I × κ, κ = 1/R
        I = (t ** 3) / 12  # 截面惯性矩
        R_min = 2 * t * (1 - crease_depth)  # 最小弯曲半径
        
        # 2. 纤维断裂判定
        strain_outer = t / (2 * R_min)  # 外层应变
        if strain_outer > material.fracture_strain:
            # 纤维断裂 → 永久变形 → 折痕成功
            fold_type = 'permanent'
        else:
            fold_type = 'elastic'  # 回弹
        
        # 3. 回弹计算
        if fold_type == 'elastic':
            # Springback ratio (经验公式)
            K = 1 - (R_min / (R_min + t))
            springback = angle_target * (1 - K)
        else:
            # 永久折痕的微小回弹
            springback = angle_target * 0.02 * (1 - crease_depth)
        
        # 4. 时间衰减 (蠕变)
        # 纸板在折叠后会继续缓慢变形
        # final_angle = target - springback * exp(-t/tau)
        tau = 3600  # 蠕变时间常数(秒)
        
        # 5. 含水率影响
        MC = material.moisture_content  # %
        MC_base = material.base_moisture  # %
        expansion = (MC - MC_base) * material.hygro_expansion_coeff
        angle_drift = expansion * 0.15  # 经验系数
        
        return FoldResult(
            final_angle=angle_target - springback + angle_drift,
            bending_radius=R_min,
            springback_deg=math.degrees(springback),
            fold_type=fold_type,
            stress_peak_mpa=E * strain_outer,
            creep_tau_s=tau,
        )
```

### 碰撞仿真: 箱体跌落测试

```python
class DropTestSimulator:
    """虚拟跌落测试 — 不用做物理样品"""
    
    def simulate(self, box_structure, drop_height_m, content_mass_kg):
        # 1. 自由落体 → 撞击速度
        v_impact = math.sqrt(2 * 9.81 * drop_height_m)
        
        # 2. 冲击力分布 (简化FEM)
        # 将盒体离散为壳单元
        elements = self.mesh_box(box_structure, element_size=5)  # 5mm网格
        
        # 3. 接触面判定
        contact_face = self.detect_contact(box_structure, drop_angle)
        
        # 4. 应力传播 (显式时间积分)
        for step in range(num_steps):
            # 外力 → 加速度 → 位移 → 应变 → 应力
            forces = self.compute_forces(elements, contact_face, v_impact)
            self.time_step(elements, forces, dt=1e-6)
        
        # 5. 破坏判定
        failures = []
        for elem in elements:
            if elem.stress > elem.material.crush_strength:
                failures.append(CrushFailure(elem))
            if elem.stress > elem.material.tear_strength:
                failures.append(TearFailure(elem))
        
        # 6. 可视化: 应力云图映射到3D
        stress_texture = self.render_stress_map(elements)
        
        return DropTestResult(
            max_stress=max(e.stress for e in elements),
            failures=failures,
            deformation_mm=max(e.displacement for e in elements),
            stress_texture=stress_texture,  # 直接贴到3D模型上
            pass_fail='PASS' if not failures else 'FAIL',
        )
```

## Round 14/20 — 可制造性实时评估

### DFM (Design for Manufacturing) 引擎

设计的同时, 系统持续评估"这个设计能不能造出来":

```python
class ManufacturabilityEngine:
    """设计时实时评估可制造性"""
    
    def evaluate(self, structure, factory_profile):
        issues = []
        
        # 1. 刀模约束
        die_board = factory_profile.die_cutter
        if structure.min_gap < die_board.min_bridge_width:
            issues.append(DFMIssue(
                severity='critical',
                msg=f'最小间距{structure.min_gap}mm < 刀桥最小宽度{die_board.min_bridge_width}mm',
                fix='增大相邻切割线间距至{die_board.min_bridge_width}mm'
            ))
        
        # 2. 折痕可行性
        for fold in structure.folds:
            if fold.angle > 170:  # 几乎对折
                if fold.material.thickness > 1.5:
                    issues.append(DFMIssue(
                        severity='warning',
                        msg=f'折痕{fold.id}角度{fold.angle}°+ 厚度{fold.material.thickness}mm → 易开裂',
                        fix='增加压痕深度或换用薄材料'
                    ))
        
        # 3. 粘合工艺
        for glue_tab in structure.glue_tabs:
            if glue_tab.area < 100:  # mm²
                issues.append(DFMIssue(
                    severity='warning', 
                    msg=f'粘口面积{glue_tab.area}mm² 不足, 粘合强度不够',
                    fix='增大粘口宽度至{math.ceil(100/glue_tab.length)}mm'
                ))
        
        # 4. 印刷约束
        if structure.total_x > factory_profile.max_sheet_width:
            issues.append(DFMIssue(
                severity='critical',
                msg=f'展开宽度{structure.total_x}mm > 最大纸幅{factory_profile.max_sheet_width}mm',
                fix='需要拆分为多片拼接, 或缩小尺寸'
            ))
        
        # 5. 拼版利用率
        utilization = self.compute_nesting(structure, factory_profile.sheet_size)
        if utilization < 0.65:
            issues.append(DFMIssue(
                severity='info',
                msg=f'拼版利用率{utilization:.0%}, 建议微调尺寸提高利用率',
                fix=self.suggest_dimensions_for_nesting(structure, factory_profile)
            ))
        
        # 6. 成本估算
        cost = self.estimate_cost(structure, factory_profile)
        
        return DFMReport(issues=issues, utilization=utilization, 
                        estimated_cost=cost, 
                        producibility_score=self.score(issues))
```

```python
class TextToBox:
    def generate(self, prompt: str):
        """
        输入: "设计一个高端化妆品礼盒, 翻盖磁铁吸合, 
              内衬EVA海绵, 外面烫金logo, 预算¥25以内"

        intent = self.llm.parse(prompt)
        # → {type: gift_box, closure: magnet, 
        #    insert: EVA_foam, finish: hot_stamp,
        #    budget: 25, product: cosmetics}
        
        # 2. 知识图谱检索
        # 化妆品 → 常见尺寸范围
        # 翻盖磁铁 → 需要书型盒结构
        # EVA内衬 → 内部需要额外深度
        knowledge = self.kg.query(intent)
        
        # 3. 生成BoxScript
        script = self.llm.generate_boxscript(intent, knowledge)
        
        # 4. 约束求解
        structure = self.solver.compile(script)
        
        # 5. 成本验证
        cost = self.dfm.estimate_cost(structure)
        if cost > intent.budget:
            # 自动优化: 减小尺寸/换材料/简化工艺
            structure = self.optimize_for_budget(structure, intent.budget)
        
        return structure
```

## Round 17/20 — 知识图谱: 包装领域的"大脑"

### PackagingKnowledgeGraph

```python
class PackagingKG:
    """包装领域知识图谱 — 所有经验的数字化"""
    
    # 节点类型
    node_types = [
        'BoxType',       # 盒型 (FEFCO 0201, 0427...)
        'Material',      # 材料 (白卡300g, E瓦楞...)
        'Process',       # 工艺 (胶印, 覆膜, 烫金...)
        'Machine',       # 机器 (Bobst 106, 海德堡...)
        'Standard',      # 标准 (FEFCO, ECMA, ISO...)
        'Product',       # 产品类型 (化妆品, 食品, 电子...)
        'FailureMode',   # 失效模式 (开胶, 塌箱, 脱色...)
        'Formula',       # 公式 (压痕宽度, 回弹角...)
        'Supplier',      # 供应商
        'CostDriver',    # 成本因子
    ]
    
    # 关系类型
    edge_types = [
        'suitable_for',    # 材料→产品
        'requires',        # 盒型→工艺
        'prevents',        # 工艺→失效模式
        'computed_by',     # 参数→公式
        'conflicts_with',  # 工艺→工艺
        'upgrades_to',     # 盒型→盒型
        'costs_more_than', # 比较关系
    ]
```

### 知识驱动的设计推理

```
用户: "我这个盒子总是在运输途中开裂"

KG推理链:
  开裂(FailureMode) 
    ←caused_by→ 压痕深度不足(Process)
    ←OR→ 纤维方向错误(Material) 
    ←OR→ 含水率变化(Environment)
  
  压痕深度不足
    ←computed_by→ 压痕宽度 = 1.5×t (Formula)
    ←requires→ 压痕刀宽度 ≥ 压痕宽度 (Constraint)
    ←check→ 当前设计压痕宽度 vs 推荐值
  
  纤维方向错误
    ←check→ 主折线方向 vs 纤维方向
    ←recommendation→ 纤维方向应垂直于主折线
  
系统输出:
  "检测到3个可能原因, 按可能性排序:
   1. 纤维方向与主折线平行(概率65%) → 建议旋转90°排版
   2. 压痕深度0.3mm不足(概率25%) → 建议增至0.45mm
   3. 运输湿度波动(概率10%) → 建议PE淋膜处理"
```

## Round 19/20 — 自进化模板引擎

### 模板不再是静态的, 而是会学习

```python
class EvolvingTemplateEngine:
    """模板从用户行为中自动进化"""
    
    def learn_from_usage(self, design_session):
        # 1. 追踪用户修改
        modifications = design_session.get_all_modifications()
        # 用户选了120370, 然后:
        #   - 把L从300改成280
        #   - 把折耳角从45°改成60°
        #   - 加了个锁底
        #   - 反复调整H在50-80之间, 最终选75
        
        # 2. 提取模式
        patterns = self.extract_patterns(modifications)
        # → "化妆品盒用户偏好: L偏小, H偏高, 锁底优于插底"
        
        # 3. 进化模板
        # 不是修改原模板, 而是创建"变体"
        variant = self.create_variant(
            base='120370',
            mutations={
                'default_corner_angle': 60,  # 原45
                'add_feature': 'auto_lock_bottom',
                'L_range': (200, 350),  # 缩窄默认范围
                'H_bias': 1.2,  # 倾向更高
            },
            context='cosmetics'  # 关联产品类型
        )
        
        # 4. A/B测试
        # 对新用户随机展示原版vs变体
        # 追踪: 完成率, 修改次数, 满意度
        self.ab_test(original='120370', variant=variant)
    
    def cross_breed(self, template_a, template_b):
        """模板杂交: 取A的结构 + B的功能 → 新模板"""
        # 例: 取翻盖盒的盖子结构 + 抽屉盒的抽屉 → 翻盖抽屉盒
        child = self.merge_structures(
            top=template_a.get_component('lid'),
            body=template_b.get_component('drawer'),
        )
        # 自动验证: 结构完整性 + 可制造性
        if self.validate(child):
            return child  # 一种全新的盒型诞生了
```
