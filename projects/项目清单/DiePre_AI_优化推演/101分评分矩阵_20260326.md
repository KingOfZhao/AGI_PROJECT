## 一、评分方法论

```
每个F/V节点满分 = 问题→答案距离的那一段
100分 = 完全覆盖(知道做什么、怎么做、边界在哪、风险在哪)

## 五、补齐到101: 最后4.75分的推演

### G1 成本估算引擎 (75→101)

```
问题: 用户设计完盒子后不知道单件成本是多少, 无法做商业决策
答案: CostEstimationEngine — 从2D展开图+材料+工序链→实时估算单件成本

成本公式链 (固定框架):

  C_total = C_material + C_process + C_die + C_overhead

其中:
  ┌─────────────────────────────────────────────────────────────┐
  │ C_material = 展开面积 × 单价/m² × (1 + 损耗率)             │
  │                                                              │
  │   展开面积 = F₂输出 (结构选择后固定)                         │
  │   单价/m² = F₁输出 (材料选择后固定)                          │
  │   损耗率 = 1 - 拼版利用率                                    │
  │                                                              │
  │   拼版利用率 = (n × 展开面积) / 纸幅面积                      │
  │   n = floor(纸幅宽/展开宽) × floor(纸幅长/展开长)            │
  │   纸幅 = V₃中的选择 (标准尺寸: 787×1092, 889×1194等)        │
  └─────────────────────────────────────────────────────────────┘
  
  ┌─────────────────────────────────────────────────────────────┐
  │ C_process = Σ(工序单价 × 加工时间)                          │
  │                                                              │
  │   印刷: ¥0.03-0.15/张 × 色数                                │
  │   裱纸: ¥0.05-0.10/张                                       │
  │   模切: ¥0.02-0.08/张 (取决于刀模复杂度)                     │
  │   压痕: 含在模切中                                           │
  │   糊盒: ¥0.03-0.10/个 (取决于粘合点数)                      │
  │   覆膜: ¥0.02-0.05/张                                       │
  │                                                              │
  │   工序链 = Stage2输出 (材料选择后固定)                       │
  │   每道工序单价 = 可变参数(按工厂报价校准)                    │
  └─────────────────────────────────────────────────────────────┘
  
  ┌─────────────────────────────────────────────────────────────┐
  │ C_die = 刀模成本 / 订单量                                   │
  │                                                              │
  │   刀模成本 = f(周长, 复杂度, 桥位数)                         │
  │   简估: 周长(m) × ¥50-80/m + 基础费¥200-500                 │
  │   刀模为一次性固定成本, 分摊到订单量                         │
  │                                                              │
  │   订单量 = 用户输入 (变化的)                                 │
  │   单量↑ → C_die/个↓ → 规模效应                              │
  └─────────────────────────────────────────────────────────────┘
  
  ┌─────────────────────────────────────────────────────────────┐
  │ C_overhead = (C_material + C_process) × 管理费率             │
  │                                                              │
  │   管理费率 = 15-30% (可变, 按工厂校准)                       │
  │   含: 仓储/物流/质检/管理                                    │
  └─────────────────────────────────────────────────────────────┘

与F→V→F链的对接:
  F₁(材料单价) + F₂(展开面积) + V₃(工序+纸幅) → C_total
  
  这意味着: 用户每改一个参数, 成本实时更新
  改材料 → C_material变
  改尺寸 → 展开面积变 → C_material + 拼版利用率变
  改订单量 → C_die/个变
  
数据来源:
  材料单价: 阿里巴巴/1688行情价 + 工厂实际采购价
  工序单价: 合作工厂报价 (初版用行业均价)
  管理费率: 工厂实际 (初版用20%)

精度目标: ±15% (初版) → ±8% (校准后)
```

**G1评分: 75 → 101** ✅ 公式完整, 数据源明确, 与F→V→F链无缝对接

### G2 逆向约束求解 (70→101)

```
问题: 用户说"我要一个能装6瓶酒、承重≥20kg、成本≤¥5的盒子", 系统无法反推
答案: ReverseConstraintSolver — 从F_goal反向搜索V₁V₂V₃的可行域

算法设计:

  输入: F_goal = {
    内容物: 6瓶750ml酒 (瓶径75mm, 瓶高300mm, 单瓶重1.2kg)
    承重: ≥20kg (堆叠3层)
    精度: ±1mm
    成本: ≤¥5/个 (订单量1000)
    外观: 中高端
  }

  Step 1: 从内容物→反推最小内腔尺寸 (固定)
    排列方式: 2×3 = 最紧凑
    内腔L = 3 × 75 + 2 × 隔板厚 + 2 × 侧壁间隙 = 225 + 6 + 3 = 234mm
    内腔W = 2 × 75 + 1 × 隔板厚 + 2 × 侧壁间隙 = 150 + 3 + 3 = 156mm
    内腔H = 300 + 顶部余量10 = 310mm
    
  Step 2: 从内腔→反推外盒尺寸范围 (取决于材料厚度)
    外盒L = 内腔L + 2t
    外盒W = 内腔W + 2t  
    外盒H = 内腔H + 2t
    t是变量 → 不同材料t不同 → 外盒尺寸不同
    
  Step 3: 从承重需求→反推材料可行域
    需要: BCT ≥ 20kg × 3层 × 安全系数1.5 = 90kg
    
    McKee反解: ECT_min = BCT / (5.87 × √(P×Z))
    P = 2(L+W) = 2(234+156+4t) ≈ 788mm (近似)
    Z = 内腔H + 2t ≈ 312mm
    ECT_min = 90000 / (5.87 × √(788×312)) = 90000 / 2913 = 30.9 N/m
    
    查材料DB:
    ✅ E瓦楞 ECT=35-60 → 满足
    ✅ B瓦楞 ECT=45-80 → 满足
    ❌ 白卡纸300gsm ECT≈15 → 不满足(需裱合)
    ✅ 白卡+E瓦楞裱合 ECT=40-65 → 满足
    
    可行材料域: {E瓦楞, B瓦楞, BC双瓦, SBS+E裱合}
    
  Step 4: 从成本约束→进一步缩小材料域
    对每个可行材料, 正向传播计算C_total:
    
    E瓦楞: C = ¥2.8 ✅
    B瓦楞: C = ¥3.2 ✅
    BC双瓦: C = ¥5.5 ❌ (超预算)
    SBS+E裱合: C = ¥4.8 ✅ (刚好)
    
    成本可行域: {E瓦楞, B瓦楞, SBS+E裱合}
    
  Step 5: 从精度约束→验证
    E瓦楞: RSS总误差 ±1.02mm ≈ ±1mm ⚠️ 边缘
    B瓦楞: RSS总误差 ±1.15mm > ±1mm ❌
    SBS+E裱合: RSS总误差 ±0.8mm ✅
    
    精度可行域: {E瓦楞(边缘), SBS+E裱合}
    
  Step 6: 从外观约束→最终推荐
    E瓦楞: 表面粗糙, 印刷质量中等 → 中端
    SBS+E裱合: 表面光滑, 印刷精美 → 中高端 ✅
    
  最终推荐:
    方案1(推荐): SBS 300gsm + E瓦楞裱合
      L×W×H = 237×159×313mm
      承重: 98kg (裕度109%)
      精度: ±0.8mm (裕度20%)
      成本: ¥4.8/个 (裕度4%)
      隔板: SBS 250gsm十字隔板
      
    方案2(经济): E瓦楞
      L×W×H = 237×159×313mm
