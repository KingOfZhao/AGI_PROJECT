# AGI v13.3 Cognitive Lattice — 调用方式与提问指南

## 一、访问方式

| 方式 | 地址 | 说明 |
|------|------|------|
| Web 可视化前端 | http://localhost:5002 | 完整交互界面，推荐使用 |
| REST API | http://localhost:5002/api/ | 程序化调用 |

## 二、对话模式（前端左上角切换）

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **Ask** | 标准问答 + 四向碰撞思维 | 通用提问、知识探索 |
| **Code** | 代码生成 + 动作引擎执行 | 写代码、生成文件、构建技能 |
| **Plan** | 规划模式，生成可执行计划 | 项目规划、任务拆解 |
| **Tool** | 工具控制器模式（LLM+Python运行时） | 复杂工具链、多步骤任务 |
| **NoCollision** | 跳过碰撞，直接对话 | 简单聊天、快速回答 |

## 四、核心 API 端点

### 4.1 对话

```bash
# 标准对话（四向碰撞）
curl -X POST http://localhost:5002/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你的问题", "mode": "ask"}'

# 停止对话
curl -X POST http://localhost:5002/api/chat/stop
```

### 4.2 节点管理

```bash
# 查看统计
curl http://localhost:5002/api/stats

# 搜索节点
curl -X POST http://localhost:5002/api/search \
  -H "Content-Type: application/json" \
  -d '{"query": "碳扩散", "limit": 10}'

# 录入人类真实节点
curl -X POST http://localhost:5002/api/ingest \
  -H "Content-Type: application/json" \
  -d '{"content": "你的知识内容", "domain": "领域名"}'

# 批量导入节点
curl -X POST http://localhost:5002/api/batch_import \
  -H "Content-Type: application/json" \
  -d '{"data": "内容1|领域1\n内容2|领域2", "format": "pipe"}'

# 节点验证（hypothesis → proven）
curl -X POST http://localhost:5002/api/node/verify \
  -H "Content-Type: application/json" \
  -d '{"node_id": 123, "status": "proven"}'
```

### 4.3 自成长

```bash
# 触发一次自成长循环（需要LLM）
curl -X POST http://localhost:5002/api/self_growth

# 触发一次离线成长（纯embedding碰撞，无需LLM）
curl -X POST http://localhost:5002/api/offline_growth

# 启动后台离线成长
curl -X POST http://localhost:5002/api/offline_growth/start \
  -H "Content-Type: application/json" \
  -d '{"interval": 300}'
```

### 4.3.1 全速推演自成长（GLM-5）

> 使用智谱GLM-5模型进行高速碰撞，聚焦热处理/材料科学、Flutter/移动端、本地模型/AGI能力三大领域。

```bash
# 单次全速推演（GLM-5, 5节点批量）
curl -X POST http://localhost:5002/api/turbo_growth \
  -H "Content-Type: application/json" \
  -d '{"batch_size": 5}'

# 启动后台全速推演（30秒间隔，5节点批量）
curl -X POST http://localhost:5002/api/turbo_growth/start \
  -H "Content-Type: application/json" \
  -d '{"interval": 30, "batch_size": 5}'

# 停止全速推演
curl -X POST http://localhost:5002/api/turbo_growth/stop

# 查看全速推演状态
curl http://localhost:5002/api/turbo_growth/status
```

### 4.4 数学公式引擎

```bash

```

```bash
curl -X POST http://localhost:5002/api/zhipu/delegate \
  -H "Content-Type: application/json" \
  -d '{"task": "解释Python GIL机制"}'

curl http://localhost:5002/api/zhipu/stats
```

### 4.7 动作引擎

```bash
# 执行高级动作
curl -X POST http://localhost:5002/api/advanced_action \
  -H "Content-Type: application/json" \
  -d '{"action": "code_synthesize", "params": {"task": "计算圆面积"}}'

# 执行Python代码
curl -X POST http://localhost:5002/api/execute \
  -H "Content-Type: application/json" \
  -d '{"code": "print(sum(range(100)))"}'
```

### 4.8 问题管理

```bash
# 查看无法处理的问题
curl http://localhost:5002/api/growth/problems

# 解决问题
curl -X POST http://localhost:5002/api/growth/problems/resolve \
  -H "Content-Type: application/json" \
  -d '{"id": "问题ID", "action": "resolve", "resolution": "解决说明"}'

# 忽略问题
curl -X POST http://localhost:5002/api/growth/problems/resolve \
  -H "Content-Type: application/json" \
  -d '{"id": "问题ID", "action": "dismiss"}'

# 一键批量自动处理所有问题（智能分类→dismiss/resolve）
curl -X POST http://localhost:5002/api/growth/problems/batch_auto
```

> **批量自动处理规则**：API超时/网络错误→dismiss | 幻觉内容(不存在的模型版本)→dismiss | 无意义内容→dismiss | 能力缺口报告→resolve | 有效技术问题→resolve | 其他→dismiss

## 七、提问技巧

1. **具体优于模糊** — "写一个Python计算碳扩散系数的函数" > "帮我算点东西"
2. **指定领域** — 模型会根据问题匹配 proven 节点，领域明确能提高匹配质量
3. **使用 Code 模式生成代码** — 会自动调用动作引擎执行验证
4. **使用 Tool 模式处理复杂任务** — 会自动规划步骤并逐步执行
5. **善用批量导入** — 可以一次性导入多个知识节点
6. **查看控制台** — 右侧控制台会显示模型的思考过程和执行步骤
