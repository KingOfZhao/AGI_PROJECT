## 从零推演：为什么能超越？

已知：
- Opus 4.6 是通用模型，无法针对特定领域深度强化
- GLM-5.1 专为代码训练，Coding Bench 45.3
- Agent 可以**实际执行代码并从结果学习**，Opus 不能
- Agent 有持久记忆（每次练习结果写入文件），Opus 每次从零开始

未知 → 目标：
- 哪些维度 GLM-5.1 目前弱于 Opus？→ 通过实际执行发现
- 哪些维度可快速追平？→ 有标准题库可验证

超越路径：深度 > 广度。每个领域打透，而非浅尝。

## 领域一：算法与数据结构（核心基础）

> 验证方法：用 Python 实现 → 提供测试用例 → 实际运行 → 所有用例通过

### 第1阶：正确性（能解决）

- [ ] **A01** 手写快速排序 + 归并排序，含边界：空数组、全相同、已排序、逆序。验证：10组测试全通过。
- [ ] **A02** 手写红黑树：插入、删除、旋转。验证：1000次随机操作后中序遍历有序。
- [ ] **A03** 手写 LRU Cache（O(1) get/put）+ LFU Cache。验证：LeetCode 146/460 测试用例。
- [ ] **A04** 实现 Trie（前缀树）+ 支持通配符搜索（`.` 匹配任意字符）。验证：LeetCode 212。
- [ ] **A05** 动态规划三题：0/1背包 + 编辑距离 + 最长公共子序列，含状态压缩版。
- [ ] **A06** 图算法：Dijkstra + Bellman-Ford + Floyd-Warshall，含负权边检测。验证：同图三算法结果一致。
- [ ] **A07** 拓扑排序（Kahn + DFS 两种），含环检测。验证：有向无环图 + 有环图各3组。
- [ ] **A08** 并查集：路径压缩 + 按秩合并，实现 Kruskal 最小生成树。
- [ ] **A09** 线段树：区间查询、区间更新（懒标记）。验证：10^6 数据 1000次操作正确性。
- [ ] **A10** 字符串算法：KMP + Rabin-Karp + Z算法，同输入对比结果。

### 第2阶：效率（快且省内存）

- [ ] **A11** 对 A01-A10 所有实现做 `timeit` 基准测试，记录 n=10^4/10^6 时的耗时。
- [ ] **A12** 将 A09 线段树用 `numpy` 向量化改写，对比纯Python版速度比。目标：≥5x加速。
- [ ] **A13** 实现跳表（Skip List），与 Python `sortedcontainers.SortedList` 对比增删查性能。

## 领域二：多语言深度（差异化优势）

> 验证方法：代码在对应语言环境中实际运行，输出预期结果

### Rust（所有权 + 性能）

- [ ] **R01** 用 Rust 实现安全的并发有界队列（不用 Mutex，用 channel）。验证：4线程并发读写无 race。
- [ ] **R02** 实现一个迷你 HTTP 静态文件服务器（仅用标准库 + `std::net`）。验证：curl 能获取文件。
- [ ] **R03** 用 Rust 实现 JSON 解析器（手写，不用 serde）。验证：解析 10 个标准 JSON 样例正确。
- [ ] **R04** Rust 零拷贝字符串处理：用 `&str` 切片分析 1GB 日志文件，内存使用 < 10MB。
- [ ] **R05** 用 Rust 的 `async/await` + `tokio` 实现并发 HTTP 请求（100个并发）。验证：延迟 p99 < 1s。

### Go（并发 + 简洁）

- [ ] **G01** 用 Go channel + goroutine 实现 Worker Pool（可配置并发数）。验证：1000任务正确完成。
- [ ] **G02** 实现 Context 超时控制：任务超时自动取消，资源正确释放（无泄漏）。
- [ ] **G03** 用 Go 实现 gRPC 服务端 + 客户端（一个 echo service）。验证：50次调用全成功。
- [ ] **G04** 用 `sync.Map` + `atomic` 实现线程安全计数器，对比 `sync.Mutex` 版本性能。

### TypeScript（类型系统）

- [ ] **T01** 用 TypeScript 条件类型 + infer 实现 `DeepReadonly<T>` 和 `DeepPartial<T>`。验证：tsc 编译通过 + 类型检查正确拒绝错误用法。
- [ ] **T02** 实现一个类型安全的 EventEmitter（事件名和参数类型完全推导）。
- [ ] **T03** 用 `zod` 实现运行时类型验证 + 从 schema 自动生成 TypeScript 类型。

### Python 高级

- [ ] **P01** 用 `asyncio` + `aiohttp` 实现带速率限制（令牌桶）的并发爬虫。验证：100 URL，不超过10 req/s。
- [ ] **P02** 实现一个 metaclass：自动为所有方法添加执行时间日志。验证：继承类方法自动计时。
- [ ] **P03** 用 `ctypes` 调用 C 动态库：写一个 C 排序函数，Python 调用并验证结果。
- [ ] **P04** 实现 `__getattr__`、`__setattr__`、`__delattr__` 的懒加载代理类。
