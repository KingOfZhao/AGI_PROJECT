以下为参考
核心架构（最高优先级实现路径）：
本地模型（通过单一或多份 .py 文件 + LangGraph 图结构构建）作为监督者节点（Supervisor Node），掌握“真实物理路径”锚点（本地推理、状态管理、结果责任）。它仅调用智谱AI API（GLM-4 系列）作为算力辅助工作者节点（Worker Node），解决自身算力/上下文限制问题。
本地模型对最终结果 100% 负责：
* 输出前必须经过自验证循环（Chain-of-Verification 或 多代理交叉校验）。
* 若结果不正确/出现幻觉 → 立即自上而下拆解为最小不可证伪子任务 → 调整调用方式（温度降低、tool_choice=auto、async 分步）→ 让智谱AI 只负责它已掌握的“真实节点”（工具调用、长上下文推理、代码生成等）。
* 同时自下而上从本地已验证节点生成新问题，寻找重叠后建立永久链接。
* 幻觉重置机制：会话上下文超过阈值或验证失败时，直接重置 LangGraph 状态（或新建 messages 列表），避免“聊久了”累积偏差。
本地 vs 智谱分工（四向碰撞重叠节点）：
* 本地（Ollama/Llama.cpp + LangGraph）：负责规划、验证、记忆管理、最终合成（隐私 + 责任）。
* 智谱API（glm-4 / glm-4-long）：负责重计算任务（工具调用、长上下文、agentic coding）。
* 重叠节点一旦建立（验证通过），即固化成“真实认知结构”，后续无需重复调用。
这正是你原描述的精炼版 + 可执行版，完全符合“自上而下拆解证伪 + 自下而上生成新问题 + 左右碰撞建立网络”的 AGI 成长机制。
