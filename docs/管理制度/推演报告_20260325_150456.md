| 层级 | 治理角色 | 代码映射 | 职责 |

### 制衡机制
- 宪法约束: L0 的 System Prompt 中植入不可逾越的'宪法'（如 Asimov Laws），事前约束而非事后审核。
- 动态淘汰: L1 统领哨根据任务成功率动态生成和解散，无能的 Orchestrator 不会被再次调用。
- 什伍连坐: L3 层级的 API 调用具备重试和降级机制，防止单点故障。

### 信息通路
方略-编组-执行流。L0(意图) -> L1(动态编组) -> L2(闭环执行) -> L3(原子动作)。去除了中间的审核环节，信息直达。
