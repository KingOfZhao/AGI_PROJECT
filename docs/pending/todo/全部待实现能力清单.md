## 数学公式编码框架 — 问题处理流程

```
用户自然语言问题
    ↓
[形式化] 匹配已知公式 + 提取变量 + 检测领域
    ↓
[拆解] 复杂问题 → 子公式树 (原子公式=proven库)
    ↓
[编码] 每个子公式 → 可执行Python lambda
    ↓
[执行] 带入变量计算 → 数值结果
    ↓
[验证] 结果合理性检查 → 注入proven节点
    ↓
强化本地模型: 下次遇到类似问题直接匹配proven公式
```
