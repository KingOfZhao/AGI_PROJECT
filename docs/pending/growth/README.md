## 🔧 核心机制

### 四向碰撞

1. **自上而下**: 大问题拆解为可证伪的小问题
2. **自下而上**: 从已知节点识别模式并构建抽象
3. **左右寻找重叠**: 跨域知识迁移
4. **重叠部分构建**: 固化为真实节点

### 节点→SKILL 转换

- **可执行性** → SKILL
- **知识性** → PCM (Proven Concept Module)
- **组合性** → Composite SKILL
- **待验证性** → HumanVerificationQueue

### 验证流程

```
生成 SKILL → 模板校验 → 依赖检查 → 可视化接入 → 调用关系分析 → 人类验证
```
