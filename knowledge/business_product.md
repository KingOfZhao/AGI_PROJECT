# 商业与产品决策框架

> 来源: 训练知识结构化提取 | 用于: 予人玫瑰项目、考试understanding/EQ题

## 1. 产品需求评估

### The Mom Test（Rob Fitzpatrick）
核心原则: **不要问人们想要什么，问他们现在在做什么。**

| ❌ 错误问法 | ✅ 正确问法 |
|------------|-----------|
| "你会用这个功能吗？" | "你上次遇到这个问题是怎么解决的？" |
| "你觉得这个想法怎么样？" | "你现在为这个问题花了多少时间/钱？" |
| "你需要这个功能吗？" | "你有没有尝试过类似的产品？" |

### 信号 vs 噪声
- **信号**: "我上周花了200块解决这个问题" → 有真实痛点
- **噪声**: "听起来不错，可能会用" → 礼貌性敷衍
- **强信号**: 用户已经付费或花时间用手动方式解决

## 2. MVP优先级

### 核心假设排序
```
1. 价值假设: 用户会为这个付费吗？→ 最先验证
2. 可行性假设: 我们能做出来吗？→ 第二验证
3. 增长假设: 用户会告诉别人吗？→ 验证后再考虑

错误做法: 先做增长功能（分享按钮）再验证价值（有人用吗？）
```

### 功能砍砍看
```
如果去掉这个功能:
- 核心流程中断 → Must have
- 体验变差但不影响使用 → Should have
- 没人注意到 → Don't build it
```

## 3. 定价策略

### 竞品定价锚定
```
锚定效应: 用户根据看到的第一个价格判断价值
策略: 先展示高价方案，再展示你的方案（看起来更划算）

具体方法:
1. 调研3-5个直接竞品定价
2. 找出价格区间
3. 你的价格 = 区间中位 × 你的差异化系数
4. 提供免费版（获客）+ 付费版（变现）
```

### 心理学定价
- 99 vs 100: 差1块但感觉便宜很多
- 年付折扣: 付10个月的钱用12个月（锁住用户）
- 免费试用期: 降低决策门槛

## 4. 用户增长框架

### AARRR漏斗
```
Acquisition（获客）→ Activation（激活）→ Retention（留存）→ Revenue（收入）→ Referral（推荐）

关键: 在前一个环节做好之前，不要优化下一个
→ 留存率 < 40% 不要做增长
→ 激活率 < 30% 不要做留存优化
```

### 北极星指标
```
好的北极星指标:
- 反映用户获得的核心价值
- 可衡量、可追踪
- 团队各职能都能影响

例子:
  Spotify: 收听时长（不是注册数）
  Slack: 每日活跃团队消息数（不是下载量）
  Airbnb: 预订天数（不是房源数）
```

## 5. 创业决策框架

### 最小可行性实验
```
假设: "花店老板需要在线预约系统"
实验: 去花店问10个老板（不是朋友）
成本: 2小时
成功标准: ≥5个老板说"我上个月因为这个损失了钱"
通过 → 做MVP
不通过 → 换方向

❌ 错误: 做了一个完整的系统才发现没人要
```

### 机会评估（3个问题）
1. **问题真的存在吗？** 用户现在怎么解决？花了多少成本？
2. **他们愿意付费吗？** 已有的解决方案他们花了多少钱？
3. **为什么是我们？** 我们有什么独特优势？（不只是"技术好"）

## 6. 予人玫瑰应用

### 当前阶段评估
```
产品: 玫瑰订阅/礼品电商
阶段: 从0到1
最关键验证: 
  1. 有人愿意在线买花吗？（价值假设）
  2. 能不能稳定交付？（可行性假设）
  3. 复购率够吗？（留存假设）

当前应避免:
  - 过度设计技术架构
  - 做太多SKU（5个精品 > 50个平庸）
  - 过早投入营销（产品没验证就推广）

应该做:
  - MVP上线，哪怕只有1个SKU
  - 追踪核心指标: 下单率、复购率、客单价
  - 和第一批用户深度沟通（The Mom Test）
```
