## 五步自验证

### Step 1 — 全闭环完整性
- [ ] 是否定义了工厂六阶段（生成→验证→发布→推广→监控→优化）？
- [ ] 后3阶段（推广/监控/优化）是否有具体实现？
- [ ] 是否标注了哪些能力已有、哪些是新增？
- **通过标准**: 6阶段定义 + 3阶段新实现 + 已有标注

### Step 2 — 推广引擎验证
- [ ] 是否支持≥3个推广渠道？
- [ ] 推广内容是否使用四向碰撞方法生成（非模板填充）？
- [ ] 是否有知乎/CSDN/X三种不同格式的模板？
- **通过标准**: ≥3渠道 + 四向碰撞 + 差异化模板

### Step 3 — 质量监控验证
- [ ] 是否定义了≥8个监控指标？
- [ ] 是否有质量告警规则（含优先级）？
- [ ] 告警是否可触发具体优化动作？
- **通过标准**: ≥8指标 + 告警规则 + 动作关联

### Step 4 — 工厂自进化验证
- [ ] RAPVL循环是否应用于工厂自身？
- [ ] Growth Score是否针对工厂指标定义？
- [ ] 参数更新是否反馈到skill-collision-engine？
- **通过标准**: RAPVL应用 + 工厂Growth Score + 参数回注

### Step 5 — 节点关系验证
- [ ] 与所有已有Skill的关系是否清晰？
- [ ] 是否标注了本Skill在进化树中的层级？
- **通过标准**: 关系清晰 + 层级明确
