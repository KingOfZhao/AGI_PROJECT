# SKILL.md - ProjectVet (项目自验证)

## Name
project-vet

## Description
项目推演/代码产出的自验证工具。基于SOUL.md中的Self-Verification Protocol，对任何重大产出执行5步验证。

## When to Use
- 完成DiePre节点推演后
- 产出新代码模块后
- 建立新公式/模型后
- 做出重要架构决策后
- 任何"我确信这是对的"时刻 → 恰恰需要验证

## Protocol

### Step 1: 逆向验证 (Red Team)
- 如果核心结论X为假，系统会出现什么矛盾？
- 找出至少1个"如果X为假，则Y必然不成立，但Y已确认成立"的链条
- 如果找不到矛盾 → 置信度+10%
- 如果找到矛盾 → 重新审视X

### Step 2: 边界测试 (Edge Case)
- 将关键参数推到极端值（最大/最小/零/负数/超大）
- 检查公式/代码在边界条件下是否:
  - 数值合理（不发散、不NaN、不溢出）
  - 物理合理（不出现负长度、超光速等）
  - 退化合理（极端条件下退化为已知简单情况）

### Step 3: 交叉验证 (Cross Check)
- 与其他独立来源的结论对比
- 如果多个独立路径通向同一结论 → 置信度+15%
- 如果存在分歧 → 标注分歧点，分析原因

### Step 4: 置信度标注
- [已验证] 实测/搜索确认 → 90%+
- [高确信] 多源一致/物理必然 → 80-90%
- [推测] 逻辑合理但缺乏直接证据 → 60-80%
- [不确定] 未知或冲突 → <60%

### Step 5: 失败模式 (Failure Modes)
- 列出3个最可能的失败原因
- 每个失败原因给出: 触发条件 + 影响 + 缓解措施

## Output Format

```
## 验证报告: [任务名]
- 时间: YYYY-MM-DD HH:MM
- 方法: [使用的方法]

### 逆向验证
✅/⚠️ [结果]

### 边界测试
✅/⚠️ [结果]

### 交叉验证
✅/⚠️ [结果]

### 失败模式
1. [失败1] → 缓解: [措施]
2. [失败2] → 缓解: [措施]
3. [失败3] → 缓解: [措施]

### 综合置信度: XX%
→ 结论状态: [已验证/高确信/推测/需补充验证]
→ 下一步: [具体行动]
```

## Auto Trigger
在AGENTS.md中配置: 重大推演/代码产出后自动执行，结果追加到VERIFICATION_LOG.md。
