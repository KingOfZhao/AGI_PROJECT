# AI4Science Bridge：让科研假设自动变成可运行代码的三螺旋框架

**——科研×代码×数学的跨领域融合，6大科学领域开箱即用**

---

## 一、痛点：90%的科研人员被代码卡住

一个材料科学的博士想验证一个新假设："某种合金的屈服强度与温度呈指数关系而非线性关系。"

他需要做什么？

1. 构建本构方程（数学建模能力）
2. 写有限元仿真代码（Fortran/C++/Python）
3. 设置参数空间扫描（实验设计）
4. 运行仿真（高性能计算）
5. 统计分析结果（p值、效应量、置信区间）
6. 撰写论文（学术写作）

**现实是：步骤2-5，他可能要花3-6个月。** 不是因为他不懂材料学，而是因为代码工程不是他的专业。同样，一个计算机科学家也不懂晶体结构，一个统计学家也不懂流体力学的Navier-Stokes方程。

问题不是任何一个单领域能力不足，而是**三个领域的知识壁垒**阻碍了科学发现的效率。

## 二、现有方案的问题

**Jupyter Notebook？** 太自由了。没有统一的实验管理，代码复用率低，6个月后自己都看不懂。

**专门的仿真软件（COMSOL/ANSYS）？** 黑箱。你输入参数，它给出结果，但你看不到底层计算逻辑，无法验证假设的特殊情况。

**AI代码助手（Copilot）？** 能写代码，但不懂科研方法论。它不会提醒你需要做对照组实验，不会检查你的统计方法是否合适。

**三者都没有解决根本问题：需要一个同时理解科研、代码和数学的统一框架。**

## 三、我们的方案：三螺旋认知模型

```
         科研假设
        ↗        ↘
代码实现 ←→ 数学建模
  ↖        ↗
    数据验证
```

这不是线性流程（假设→代码→结果），而是**螺旋上升**的循环：

**螺旋1：科研→代码** — 把科研假设翻译成可运行的实验代码。例如："指数关系"翻译为有限元网格+本构方程+边界条件+求解器。

**螺旋2：代码→数学** — 验证代码实现的数学正确性。例如：网格收敛性测试、解析解对比、数值稳定性分析。

**螺旋3：数学→数据** — 理论预测 vs 实际数据的对比。例如：回归分析、残差诊断、模型选择（AIC/BIC）。

**螺旋4：数据→科研** — 实验结果修正或推翻原假设。例如：数据不支持指数关系？可能是幂律关系，或者存在相变点。

每一圈螺旋都让认知更深入一步。这就是SOUL哲学中"人机闭环"在科研场景的具现化。

## 四、框架细节：SOUL五律×AI4Science

### 1. 已知/未知分离

```
已知（每个实验开始前必须明确）:
  - 理论假设（什么关系？什么机制？）
  - 可用数据（实验数据/公开数据集/文献数据）
  - 计算资源（CPU/GPU/集群/云）
  - 验证标准（统计显著性/效果大小/可复现性）

未知（实验要探索的）:
  - 模型参数的最优值
  - 假设在什么条件下成立/不成立
  - 与竞争理论的比较
  - 外推到新条件的预测能力
```

### 2. 四向碰撞应用于实验设计

```
正面: 这个实验设计能否直接验证假设？
反面: 什么情况下这个实验会给出假阳性/假阴性？
侧面: 这个实验的方法能否迁移到其他领域？
整体: 这个实验在整个研究路线中的位置是什么？
```

### 3. 人机闭环

```
AI负责: 文献聚合、数据清洗、代码生成、统计计算、可视化
人类负责: 提出创造性假设、物理直觉判断、异常结果解释、论文撰写
循环: AI出初步结果 → 人类质疑 → AI改进方法 → 人类确认
```

### 4. 文件即记忆

```
ai4science_bridge/
  projects/{project_name}/
    hypothesis.md           — 假设描述及置信度
    experiment_design.md    — 实验设计（四向碰撞记录）
    code/                   — 实验代码（版本控制）
    results/                — 原始结果（数据+图表）
    analysis/               — 统计分析报告
    lessons_learned.md      — 本次实验的教训
```

### 5. 红线

```
🔴 不使用黑箱AI模型输出科研结论（必须可解释）
🔴 不伪造或选择性报告实验数据
🔴 不忽略模型适用边界（外推必须有警告）
🔴 可复现性是最低标准（固定随机种子、记录环境）
🔴 代码必须有测试（单元测试+集成测试+回归测试）
```

## 五、六大领域开箱即用

我们为当前最热门的6个AI4Science领域预置了完整的知识模板：

### 1. 分子模拟

```
工具链: RDKit + ASE + OpenMM + GROMACS
核心知识: 力场模型(UFF/MMFF/OPLS)、分子动力学、自由能计算
验证方法: 结合能与实验IC50对比、构象搜索覆盖率
跨领域: →医疗(药物设计) →工业(材料性质) →代码(HPC优化)
```

### 2. 气候建模

```
工具链: xarray + Dask + cftime + cartopy
核心知识: GCM降尺度、极端事件统计、ENSO周期
验证方法: CRPS评分、Brier评分、空间相关性
跨领域: →农业(产量预测) →政策(碳排放) →金融(天气衍生品)
```

### 3. 计算流体力学

```
工具链: OpenFOAM + PySPH + FEniCS
核心知识: Navier-Stokes方程、湍流模型(k-ε/LES)、网格生成
验证方法: 网格收敛性、与解析解/实验数据对比
跨领域: →工业(气动优化) →医疗(血流模拟) →农业(通风设计)
```

### 4. 生物信息学

```
工具链: Biopython + Scanpy + scikit-allel + snakemake
核心知识: 序列比对、基因表达分析、GWAS、通路富集
验证方法: FDR校正、效应量、独立队列验证
跨领域: →医疗(精准医疗) →代码(生物信息管道) →农业(基因组育种)
```

### 5. 材料科学

```
工具链: pymatgen + ASE + VASP接口 + matminer
核心知识: 晶体结构、DFT计算、相图、性能预测
验证方法: 形成能、带隙、弹性常数与实验对比
跨领域: →工业(新材料开发) →能源(电池材料) →环境(催化剂)
```

### 6. 天体物理

```
工具链: astropy + healpy + emcee(UK) + lightkurve
核心知识: 光谱分析、恒星演化模型、引力波信号处理
验证方法: χ²拟合、贝叶斯证据、交叉验证
跨领域: →科研(基础物理) →代码(信号处理) →教育(科学传播)
```

## 六、与传统方案的对比

| 维度 | 传统方式 | AI4Science Bridge |
|------|---------|-------------------|
| 假设→代码 | 手动3-6个月 | AI辅助生成，1-2周 |
