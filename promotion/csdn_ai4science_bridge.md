# AI4Science Bridge：科研、代码、数学的三领域融合框架

## 为什么需要这个框架？

科学发现的瓶颈已经从"理论不够"转移到了"计算不够"。

一个典型的科研流程：
1. 提出假设（需要物理/化学/生物直觉）
2. 设计实验（需要数学建模能力）
3. 编写代码（需要软件工程能力）
4. 运行验证（需要统计推断能力）
5. 发表论文（需要学术写作能力）

**90%的科研人员不擅长第3步（代码），50%不擅长第4步（统计）。**

## 我们的解决方案

`ai4science-bridge` 把科研→代码→数学画成一个三螺旋：

```
    科研假设
   ↗        ↘
代码实现 ←→ 数学建模
  ↖        ↗
    数据验证
```

四个螺旋循环：
- 科研→代码：假设编码为可运行实验
- 代码→数学：算法理论基础验证
- 数学→数据：理论预测 vs 实际数据
- 数据→科研：实验结果修正假设

## 六大热门领域开箱即用

1. 分子模拟（RDKit/ASE）
2. 气候建模（xarray/Dask）
3. 计算流体（OpenFOAM/PySPH）
4. 生物信息（Biopython/Scanpy）
5. 材料科学（pymatgen/ASE）
6. 天体物理（astropy）

## 红线

不搞黑箱AI、不伪造数据、不忽略模型边界、可复现是最低标准。
